def pose_from_distance(initial_estimate: np.array,
                       centers: np.array,
                       distances: np.array,
                       iter_limit: int) -> np.array:
    '''
    initial_estimate -- initial estimate of the robot's position
    centers -- beacon center positions (as known from the map)
    distances -- a list of distances to the four beacons
    iter_limit -- maximum number of allowed iterations
    '''
    x,y,z = sp.symbols('x, y, z')
    pose = list()

    for i in range(len(distances)):
        
        initial = np.array([initial_estimate[0],initial_estimate[1],initial_estimate[2]] )
       
        d1 = distances[i,0]
        d2 = distances[i,1]
        d3 = distances[i,2]
        d4 = distances[i,3]
        
        g1 = ( (centers[0,0]-x)**2 + (centers[0,1]-y)**2 + (centers[0,2]-z)**2 - d1**2)
        g2 = ( (centers[1,0]-x)**2 + (centers[1,1]-y)**2 + (centers[1,2]-z)**2 - d2**2)
        g3 = ( (centers[2,0]-x)**2 + (centers[2,1]-y)**2 + (centers[2,2]-z)**2 - d3**2)
        g4 = ( (centers[3,0]-x)**2 + (centers[3,1]-y)**2 + (centers[3,2]-z)**2 - d4**2)
        
        Gx = sp.Matrix([[g1],[g2],[g3]])
        Y = sp.Matrix([x, y, z])
        jac_Gx = Gx.jacobian(Y)
       
        jac_inv = jac_Gx.inv()
        
        multi_dim = jac_inv*Gx
        
        initial_curr = initial
        for n in range(iter_limit):
            initial[0] = initial[0] - multi_dim[0].evalf(subs = {x:initial_curr[0], 
                                                                 y:initial_curr[1], 
                                                                 z:initial_curr[2]})
            initial[1] = initial[1] - multi_dim[1].evalf(subs = {x:initial_curr[0], 
                                                                 y:initial_curr[1], 
                                                                 z:initial_curr[2]})
            initial[2] = initial[2] - multi_dim[2].evalf(subs = {x:initial_curr[0], 
                                                                 y:initial_curr[1], 
                                                                 z:initial_curr[2]})
            initial_curr = initial
        pose.append(initial)
      
    return np.array(pose)

Following are the equations

Tip of Link 1:

$x_{1} = L_{1}cos\theta_{1}$

$y_{1} = L_{1}sin\theta_{1}$

where,

$L_{1}$ is the length of link 1
$\theta_{1}$ is the angle subtented by the link1 with x-axis

$x_{2} = L_{1}sin\theta_{1} + L_{2}sin(\theta_{1}+\theta_{2})$

$L_{1}$ is the length of the link 2
$\theta_{2}$ is the angle subtended by the link 2 with the extended link 1 axis and $x_{2}$

After the $37^{th}$ iteration, the value of theta we obtain is a bit different than previous ones.So it causes the unnatural shift of joint.As per the reference, this might be because of the concept called as "Internal Singularities" (also known as joint space singularities). They are generally caused by an alignment of the robots axes in space. For example, if 2 axes become aligned in space, rotation of one can be canceled by counterrotation of the other, leaving the actual joint location indeterminate. Also, certain kinematic alignments specific to each manipulator can cause these.

From observation we can say that after $37^{th}$ iteration the axes align themselves.

Reference- http://www.cs.columbia.edu/~allen/F15/NOTES/jacobians.pdf
